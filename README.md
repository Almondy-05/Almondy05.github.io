# 短線戰情室 Trade Analyzer（網頁版）

讀取證券對帳單 CSV，自動做 FIFO 配對，算出短線交易績效與資金效率指標。
這是桌面版（Eel + PyInstaller 打包成 .exe）的純前端網頁重寫版本，邏輯完全
對應，不需要安裝任何東西，手機或電腦直接用瀏覽器開就能用。

## 這個版本解決了什麼問題

桌面版只能在 Windows 上跑。這個版本把 `core/` 底下五個 Python 模組
（`models.py`、`parser.py`、`matcher.py`、`capital.py`、`stats.py`）跟
`index_data.py` 的邏輯全部用 JavaScript 重寫一份，搭配原本就是 HTML/JS
的前端畫面，整包變成一個不需要後端伺服器、純瀏覽器執行的網頁。iOS、
Android、桌面瀏覽器都能開，不需要打包成 APP 或 APK。

所有運算都在使用者自己的瀏覽器裡完成，CSV 內容不會被送到任何伺服器。

這一版對齊最新桌面版，除了原本的核心績效跟資金效率指標，另外有：風險調整
後報酬（夏普值、索提諾比率，用資金占用日報酬率計算、未年化）、單筆期望值、
進出場日相對 6 日均線的乖離率（BIAS）。大盤資料抓取改成手動觸發，並新增
「清除大盤快取」。

## 檔案結構

```
trade_analyzer_web/
├── index.html            主畫面
├── style.css              深色交易終端機風格（沿用桌面版設計），加了手機版排版調整
├── app.js                  畫面渲染、檔案選擇流程協調、canvas 圖表繪製
├── core/
│   ├── dateUtil.js          日期工具：用 UTC epoch day 整數運算，避免時區誤差
│   ├── numberUtil.js        pyRound()：用十進位字串運算還原 Python round() 的
│   │                          「四捨五入到偶數」規則，連浮點數邊界案例都對得上
│   ├── models.js             對應 models.py
│   ├── parser.js             對應 parser.py，含通用 CSV tokenizer（支援引號欄位）
│   ├── matcher.js            對應 matcher.py，FIFO 配對引擎
│   ├── capital.js            對應 capital.py，資金效率計算
│   ├── stats.js               對應 stats.py，統計彙總（輸出沿用跟 Python 版本
│   │                          完全相同的中文鍵名，畫面渲染邏輯才能照搬不用改）
│   ├── indexData.js          對應 index_data.py，改用瀏覽器 fetch() + localStorage
│   └── encoding.js           檔案編碼偵測，改用 TextDecoder 取代 Python 的
│                              utf-8-sig/cp950/big5 多重嘗試
└── favicon.png
```

所有 JS 檔案都用傳統 `<script src="...">` 載入，沒有用 ES module。這是
刻意的選擇：用 `file://` 直接開啟本機 HTML 檔案時，部分瀏覽器（尤其
Chrome）會擋掉 `type="module"` 的載入，傳統 script 標籤沒有這個限制，
確保「下載下來直接點開」這個最基本的使用情境一定能動。

## 怎麼用

**最簡單：直接開啟檔案**
把整個 `trade_analyzer_web` 資料夾下載下來，雙擊 `index.html`，電腦上的
瀏覽器會直接打開。Android 手機也可以：把資料夾傳到手機（例如用雲端硬碟
同步），用檔案管理 App 點開 `index.html`，選「用瀏覽器開啟」。

**iPhone 的話建議架站**
iOS Safari 對於直接瀏覽本機資料夾裡的網頁檔案限制比較多，不像 Android
那麼方便。最簡單的解法是把這個資料夾上傳到一個免費的純靜態空間，例如
GitHub Pages、Netlify、Cloudflare Pages，產生一個網址後用手機瀏覽器
直接開那個網址，效果跟本機開啟完全一樣（純前端、沒有後端，靜態空間
就足夠了），而且之後要用都只要記住一個網址，比每次找檔案方便。

## 跟桌面版的差異

**大盤指數連網抓取改成手動觸發，會撞到 CORS**

桌面版用 Python 的 `urllib` 直接連到證交所，沒有跨來源限制。瀏覽器有同源
政策，證交所端點如果沒有主動開放跨來源請求（`Access-Control-Allow-Origin`），
瀏覽器會直接擋下，JS 只看得到籠統的網路錯誤。

對齊最新桌面版，抓取大盤資料是手動的，不會在分析的時候自動連網：分析永遠
只讀取目前快取裡已經有的資料，缺漏的部分讓對應欄位顯示空白，不會因為連線
狀況不穩定，讓同一份對帳單每次跑出不一樣的速度或結果。要補大盤資料有三條路：

1. **上傳大盤指數 CSV**（證交所「發行量加權股價指數歷史資料」頁面下載），
   一定能用，不受 CORS 影響，是最可靠的方式。
2. **「立即抓取大盤資料」按鈕**：分析完之後按這個按鈕，才會連網補齊缺漏的
   日期。批次抓取用「整月」為單位的端點（跟上傳 CSV 對應同一份資料），
   把需要的日期按年月分組，一個月份只打一次請求。能不能成功取決於證交所
   對跨來源請求的回應，連不到就改用上傳 CSV。
3. **「測試大盤連線」按鈕**：單獨測一次連線，把實際錯誤原因顯示出來。

需要修正或重新整理大盤資料的時候，可以先按**「清除大盤快取」**清乾淨，
再重新上傳或抓取，不會有舊資料殘留。

漲跌百分比一律用「相鄰兩筆」計算，不管收盤指數是連網抓的還是上傳 CSV
來的都走同一套算法。早期版本曾經讓連網抓到的直接採用官方漲跌百分比、
上傳 CSV 的才用相鄰兩筆推算，兩種算法有捨入精度差異，同一份分析混到兩種
來源時大盤基準會兜不起來。統一成一套算法之後結果一致。

**本機快取改用 localStorage**

桌面版存在 `%APPDATA%/TradeAnalyzer/index_cache.json`。網頁版改存在
瀏覽器的 `localStorage`，綁定在開啟這個頁面的網址（或本機檔案路徑）上，
同一個瀏覽器下次打開還在，邏輯跟桌面版一致；差別是清瀏覽器資料、換
瀏覽器、或用無痕模式的話快取不會保留，這點桌面版的檔案快取沒有這個
限制。

**檔案編碼偵測改用 TextDecoder**

桌面版依序嘗試 `utf-8-sig` → `cp950` → `big5` → `utf-8`。瀏覽器的
`TextDecoder` 沒有 `cp950`，但 WHATWG Encoding 標準規定瀏覽器都要支援
`big5`，對台灣券商 CSV 的實際涵蓋範圍幾乎一致，足以取代。

**Python `round()` 的四捨五入到偶數規則，用十進位字串運算還原**

JS 原生沒有對應 Python `round()` 的函式。`numberUtil.js` 的 `pyRound()`
直接取出該數字的高精度十進位字串表示，對字串本身做進位判斷，避開
「乘以 10 的 n 次方再四捨五入」這種寫法在乘法過程中額外產生的浮點誤差，
正確處理「這個數字的二進位浮點數實際存的值，比看起來的十進位數字略大或
略小一點點」這種邊界情況：例如 `0.945` 在二進位浮點數裡實際存的是
`0.9450000000000001`，正確的捨入結果是 `0.95`；如果誤判成「剛好卡在
.5」，會錯誤觸發四捨五入到偶數，變成 `0.94`。已經用涵蓋這類邊界案例的
測試驗證過，跟 Python 的 `round()` 逐一比對結果一致。


## 已經驗證過的東西

JS 版本的每個函式輸出都拿去跟桌面版的 Python 原始碼（同一份 `core/`
邏輯）逐項比對數字，確認完全一致：

- 用 `sample_test.csv` 跑過完整流程（解析、FIFO 配對、統計彙總、資金
  水位），JS 輸出的每一個欄位都跟 Python 版 `test_engine.py` 的輸出
  逐項比對過，數值完全一致。
- 另外設計了一組更完整的合成測試資料，涵蓋可疑配對（跨日放空）、
  期末未平倉、多檔案合併去重（含委託書號跟備用 key 兩種去重路徑）、
  月彙總、效益比 Top N 排序、大盤對應分析、大盤基準損益曲線，JS 跟
  Python 兩邊各自跑一次完整流程，輸出的 JSON 逐欄位比對，完全一致。
- `pyRound()` 額外用一組已知會暴露浮點數邊界問題的案例測試過（例如
  `2.675`、`0.125`、整數邊界的 `.5`），跟 Python `round()` 逐一比對
  結果一致。
- 用無頭瀏覽器（Chromium）實際開啟 `index.html`、模擬選擇 CSV 檔案，
  完整驗證從檔案選擇到畫面渲染的整個流程：手機尺寸視窗下 header 不會
  橫向溢出、canvas 圖表確實畫出內容、大盤指數 CSV 上傳後「大盤對應
  分析」「大盤基準疊圖」正確顯示數字、重新整理頁面後 localStorage
  快取正確保留、CORS 被擋的情境下分析流程不會中斷或拋出例外、Big5
  編碼的 CSV 檔案正確解碼成可讀的繁體中文。

## 已知限制

- 大盤指數連網抓取在大多數情況下會因為 CORS 被瀏覽器擋下，上傳 CSV
  是目前比較可靠的取得方式。
- `localStorage` 的容量上限通常是 5–10MB，對這個用途（逐日大盤指數，
  一年大約 250 筆資料）綽綽有餘，不會碰到上限。
- 沒有做任何資料壓縮或最佳化，CSV 筆數落在個人短線交易者一般的量級
  （幾十到幾百筆）完全沒有效能問題；筆數如果到幾萬筆以上量級，FIFO
  配對引擎目前用陣列模擬佇列（`shift()`），效能會明顯下降，但這個
  量級已經遠超出這個工具設計的單一使用者自用情境。

## 沒有改動的部分

核心邏輯規則（FIFO 配對、可疑配對偵測、資金效率公式、凹單率、效益比
排序、賣飛指標、大盤基準曲線）全部原封不動沿用桌面版的定義，細節說明
請參考桌面版 README 的「各項指標的計算公式」「計算邏輯說明」兩節，
網頁版的算法完全相同，這裡不重複寫一份。畫面視覺風格（深色背景、
等寬字體、紅漲綠跌的台股慣例、琥珀色強調色）也原封不動沿用。
